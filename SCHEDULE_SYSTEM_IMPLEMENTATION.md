# Schedule System Implementation Guide

## Overview

The schedule system has been completely refactored to use real data from the database. The system now displays live lecture sessions with full support for both students and teachers.

## Architecture

### 1. **Schedule Service** (`lib/schedule-service.ts`)

Core service for fetching and managing schedule data.

#### Key Functions:

```typescript
// Fetch student schedule - gets all sessions for enrolled sections
getStudentSchedule(studentId, universityId, startDate, endDate)
  → Returns: { sessions: LectureSessionExtended[], error }

// Fetch teacher schedule - gets all sessions where instructor is assigned
getTeacherSchedule(instructorId, universityId, startDate, endDate)
  → Returns: { sessions: LectureSessionExtended[], error }

// Subscribe to TOTP updates for real-time codes
subscribeTOTPUpdates(lectureSessionId, onUpdate)
  → Returns: Supabase channel for real-time listening

// Get TOTP session data
getTOTPSession(lectureSessionId)
  → Returns: { session: TOTPSession, error }
```

#### Helper Functions:

- `formatTime()` - Convert HH:mm:ss to 12-hour format
- `calculateDuration()` - Get class duration in minutes
- `getStatusColor()` - Get status badge color
- `getStatusLabel()` - Get human-readable status

### 2. **Session Block Component** (`components/SessionBlock.tsx`)

Reusable block component for displaying individual lecture sessions.

**Features:**
- Course title and status badge (Upcoming/Ongoing/Completed)
- Time, location, instructor information
- Special class indicator
- TOTP active indicator
- Late policy information
- Smooth animations with Reanimated

**Usage:**
```tsx
<SessionBlock
  session={lectureSession}
  isFirstItem={index === 0}
  colors={colors}
  index={index}
/>
```

### 3. **ViewScheduleScreen** (`screens/ViewScheduleScreen.tsx`)

Main schedule display component with dual-view support.

**Features:**
- **Daily View**: Shows sessions by date with date picker navigation
- **Weekly View**: Shows all sessions for the current week
- **Dual Role Support**: Automatically adapts for student or teacher
- **Real-time Data**: Fetches from database on component mount
- **Loading States**: Shows spinner while loading
- **Error Handling**: Displays error messages if fetch fails

**Data Flow:**
1. Component mounts → Check user role and profile
2. Fetch schedule data for 30 days past + 90 days future
3. Convert LectureSessionExtended to ScheduleClass format
4. Display in selected view mode (daily/weekly)

## Data Models

### LectureSessionExtended
```typescript
{
  id: UUID
  university_id: UUID
  timetable_id: UUID | null
  course_id: UUID
  section_id: UUID
  room_id: UUID | null
  instructor_ids: UUID[]
  session_date: string (YYYY-MM-DD)
  start_time: string (HH:mm:ss)
  end_time: string (HH:mm:ss)
  session_status: 'scheduled' | 'active' | 'completed' | 'cancelled' | 'rescheduled'
  totp_required: boolean
  is_special_class: boolean
  is_active: boolean
  is_cancelled: boolean
  
  // Nested data
  course?: { id, code, name }
  room?: { id, name, building?: { name } }
  instructors?: [{ id, name }]
}
```

### TOTPSession
```typescript
{
  id: UUID
  lecture_session_id: UUID
  code: string (6-digit code)
  totp_mode: 'autogenerated_dynamic' | 'autogenerated_static' | 'manual'
  generated_at: timestamp
  expires_at: timestamp (30-second expiry)
  last_refresh_at: timestamp | null
}
```

## Student Schedule Flow

1. **Fetch Student Enrollments**: Get sections from `student_enrollments` table
2. **Fetch Lecture Sessions**: Get all sessions for those sections
3. **Populate Course/Room/Instructor**: Join with related tables
4. **Filter by Date**: Show only sessions within requested date range
5. **Display**: Use SessionBlock component for each session

### Key Queries:
```sql
-- Get student's sections
SELECT section_id FROM student_enrollments 
WHERE student_id = ? AND is_active = true

-- Get lecture sessions for sections
SELECT * FROM lecture_sessions 
WHERE section_id IN (?, ?, ...) 
  AND session_date BETWEEN ? AND ?
  AND is_active = true 
  AND is_cancelled = false
ORDER BY session_date, start_time
```

## Teacher Schedule Flow

1. **Get Instructor ID**: From `instructor` in AuthContext
2. **Fetch Lecture Sessions**: Get all sessions where instructor_ids contains this instructor
3. **Populate Course/Room Data**: Join with related tables
4. **Filter by Date**: Show only sessions within requested date range
5. **Display**: Use SessionBlock component for each session

### Key Queries:
```sql
-- Get lecture sessions for instructor
SELECT * FROM lecture_sessions 
WHERE instructor_ids @> ARRAY[?]::uuid[] 
  AND session_date BETWEEN ? AND ?
  AND is_active = true 
  AND is_cancelled = false
ORDER BY session_date, start_time
```

## Authentication Integration

The system uses authentication context to determine:

- **User Role**: `isStudent` or `isTeacher`
- **User ID**: For student schedule
- **Instructor ID**: For teacher schedule (from `instructor` context)
- **University ID**: For filtering by university

```typescript
const { userProfile, isStudent, isTeacher, instructor } = useAuth();

// For students: use userProfile.id
// For teachers: use instructor.id
```

## Performance Optimization

1. **Date Range Caching**: Fetches 30 days past + 90 days future (minimizes refetches)
2. **Selective Joins**: Only fetches necessary course/room/instructor data
3. **Array Subscriptions**: Uses PostgreSQL array operations for efficient instructor filtering
4. **Reusable Components**: SessionBlock component with memoization
5. **Lazy Loading**: Data fetched on component mount, not on every selection

## Error Handling

The system handles multiple error scenarios:

1. **No User Profile**: Shows loading state until profile loads
2. **Teacher Without Instructor**: Shows error message
3. **Fetch Errors**: Displays user-friendly error message
4. **No Classes**: Shows "No classes scheduled" message
5. **Invalid Date Range**: Gracefully handles edge cases

## Troubleshooting

### Schedule not loading
1. Check that user profile is fully loaded
2. Verify user has valid enrollments (for students) or instructor record (for teachers)
3. Check browser console for API errors
4. Verify Supabase connection

### Wrong sessions showing
1. Check student enrollments are correct
2. Verify lecture_sessions have correct section_id
3. For teachers, ensure instructor_ids array contains the instructor

### UI issues
1. Clear app cache and reload
2. Check that colors object is properly passed
3. Verify SessionBlock component imports
